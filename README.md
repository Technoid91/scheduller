# Scheduler — Shared House Duty Manager

Веб-приложение для составления графиков дежурств в shared house. Генерирует PDF для печати и отправляет его в Telegram.

---

## Быстрый старт

### 1. Настройте окружение

Скопируйте `.env.example` в `.env` и заполните нужные значения:

```bash
cp .env.example .env
```

### 2. Установите зависимости

```bash
pip install -r requirements.txt
```

### 3. Запустите

```bash
python app.py
```

Приложение откроется по адресу, указанному в `.env` (по умолчанию `http://localhost:5000/scheduler`).

---

## Файл .env

Все настройки окружения хранятся в `.env` в корне проекта. Файл не попадает в git.

| Переменная | Обязательная | Пример | Описание |
|---|---|---|---|
| `TELEGRAM_TOKEN` | для Telegram | `123456:ABC-xxx` | Токен бота, полученный у [@BotFather](https://t.me/BotFather) |
| `TELEGRAM_CHAT_ID` | для Telegram | `123456789` или `111,222,333` | ID чата или несколько через запятую. Узнать у [@userinfobot](https://t.me/userinfobot) |
| `PORT` | нет | `5001` | Порт на котором запускается приложение. По умолчанию `5000` |
| `URL_PREFIX` | нет | `/scheduler` | URL-префикс приложения. Используйте `/` если префикс не нужен |

Пример заполненного `.env`:

```
TELEGRAM_TOKEN=123456789:ABCdefGhIJKlmNoPQRsTUVwxYZ
TELEGRAM_CHAT_ID=123456789,987654321
PORT=5001
URL_PREFIX=/scheduler
```

## Возможности

### График дежурств

- Непрерывная последовательность дежурств задаётся через **точку отсчёта** — дату и комнату, которая дежурит в этот день. Скрипт строит расписание в обе стороны от этой точки для любого месяца и года.
- Настраиваемое **количество дней на комнату** (по умолчанию 2).
- Навигация по месяцам стрелками — можно смотреть расписание на любой месяц.
- Два вида отображения: **список** и **календарная сетка**.

### Комнаты

- Произвольный список комнат с произвольной нумерацией.
- **Drag & drop** для изменения порядка дежурств.
- Кнопка **«выкл/вкл»** — исключить комнату из расписания (например, если жилец временно отсутствует) и вернуть обратно. Последовательность при этом пересчитывается автоматически.

### Снапшоты (история конфигураций)

Изменения в расписании хранятся в виде снапшотов — каждый снапшот фиксирует состояние настроек начиная с определённого месяца.

- Снапшот создаётся **явно по кнопке** — случайных сохранений нет.
- Для каждого месяца автоматически применяется **ближайший предыдущий снапшот**. Например, если снапшоты есть за январь и май, то февраль–апрель используют январский, июнь и далее — майский.
- Снапшоты хранятся в `config.json` в **хронологическом порядке**.
- Снапшот можно удалить — месяц вернётся к предыдущему снапшоту.
- Индикатор у каждого месяца показывает: **✦ свой снапшот** / **← откуда унаследован** / **дефолт**.

### PDF

- Генерируется одностраничный PDF формата A4 на украинском языке.
- Высота строк фиксирована под 31 день — в более коротких месяцах остаётся пустое место внизу.
- Встроенный шрифт DejaVu Sans (лицензия свободная, коммерческое использование разрешено) — работает без системных шрифтов на любой ОС.
- Три колонки: **Дата** (жирный), **Кімната** (обычный), **Підпис** (пустая, для ручных отметок).

### Telegram

- Отправка PDF одной кнопкой.
- Поддержка **нескольких получателей** — укажите несколько chat_id через запятую в `.env`.
- **Автоотправка** — если включена галочка в интерфейсе, бот сам отправит график на следующий месяц в последний понедельник текущего месяца в 10:30. Если последний понедельник совпадает с последним днём месяца — отправка переносится на неделю раньше.

│   └── DejaVuSans-Bold.ttf
└── templates/
    └── index.html        # Веб-интерфейс (SPA на vanilla JS)

## Автозапуск

### macOS / Linux (launchd / cron)

```bash
# Добавить в crontab
@reboot cd /path/to/scheduler && python app.py >> scheduler.log 2>&1
```

### Windows

Создайте `start.bat`:
```bat
cd C:\path\to\scheduler
python app.py
```
Добавьте ярлык в `%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`.

## config.json

Создаётся автоматически при первом запуске. Не редактируйте вручную. Структура:

```json
{
  "rooms": [1, 2, 3],
  "autosend": false,
  "snapshots": {
    "2026-01": {
      "order": [1, 2, 3],
      "active": [1, 2, 3],
      "days_per_room": 2,
      "anchor_date": "2026-01-01",
      "anchor_room": 1
    }
  }
}
```
